## 1. 目标

1. **删除死抽象层**：移除被 §IX supersede 的 `CnuiFormAdapter` / `FormRegistry` / `register-form.ts` 三件套。
2. **habits 手写化**：`HabitCreationCard` 改为 tasks 同款手写范式（弃 adapter、直引 `HabitForm`、`serverErrors` 直传），顺带接上当前断掉的 habits create 回填管线（Lane B 漏网）。

4. **新增 validator 前向守卫**：在 `validate-domain-structure.ts` 落地 L4-1（禁 `CnuiFormAdapter`）+ L7-2（禁 `FormRegistry.register`/`register-form.ts` 残留），防止 re-introduction。

## 3. 背景：三个关键现状（不可从代码一眼看出）

### 3.1 `fieldMapping` 全是恒等映射（no-op）
`register-form.ts` 的 `fieldMapping` 每一条都是 `title: 'title'`、`defaultTime: 'defaultTime'`……因此 `CnuiFormAdapter` 的 `mapDataToForm`/`mapFormToData` 与 `intent.ts:1319-1329` 的 `mappedFields` 循环，输出恒等于输入。`intent.ts:1319` 注释"保留下层映射逻辑"保留的是一段空操作。

### 3.2 `HabitForm` 自带全部默认值
`habit-form.tsx:90-103` 用 `useState(initial?.defaultTime ?? "07:00")` 等 fallback 内建了与 `register-form.ts` `defaults` **逐字相同**的默认值：
```
defaultTime "07:00" / earliestTime "06:30" / latestStartTime "08:00" /
defaultDuration 30 / minDuration 15 / trackable true /
frequencyType "daily" / daysOfWeek [1,2,3,4,5]
```
因此 `register-form.ts` 的 `defaults` 是纯冗余——删除后传 `initial={dataModel}`（仅含 `{startDate}`），`HabitForm` 自身 fallback 产出完全相同的初值。**"默认值搬去哪"不构成问题。**

### 3.3 habits create 回填当前是断的（Lane B 漏网）
- `HabitCreationCard` 持有一个永远空 `[]` 的 `useState<string[]>`，且其 props 接口**根本未声明 `serverErrors`**。
- `CnuiFormAdapter` 把 `serverErrors` 渲染成表单**下方一个独立红框**（`cnui-form-adapter.tsx:96-102`），**从不传入** `HabitForm`。
- 而 `HabitForm` 内部早已把 `useServerErrorBackfill` 全接好（`habit-form.tsx:107` + 各字段 `serverFieldErrors` 渲染）。

结论：迁移时把 `serverErrors` 直传 `HabitForm`，既完成手写化、又顺手接上断点——一举两得，且行为仅"从断→通"，无回退风险。

## 5. 设计

### 5.A `HabitCreationCard` 手写化（改 1 文件）

弃 `CnuiFormAdapter`，直引 `HabitForm`，接口补 `serverErrors` 并直传：

```tsx
'use client'
import { HabitForm } from '../../components/habit-form'
import type { HabitFormFields } from '../../components/habit-form'

interface HabitCreationCardProps {
  surfaceType: string
  dataModel: Record<string, unknown>
  /** 框架契约，保留；当前不逐键回传（与旧 adapter 行为一致） */
  onDataChange: (data: Record<string, unknown>) => void
  onConfirm: (data: Record<string, unknown>) => void
  onCancel: () => void
  isLoading?: boolean
  isDone?: boolean
  /** CnuiRenderer 已透传；旧接口缺失是回填断的根因 */
  serverErrors?: string[]
}

export function HabitCreationCard({ dataModel, onConfirm, onCancel, isLoading, serverErrors }: HabitCreationCardProps) {
  return (
    <HabitForm
      initial={dataModel as Partial<HabitFormFields>}
      onSubmit={onConfirm}
      onCancel={onCancel}
      isLoading={isLoading}
      serverErrors={serverErrors}
    />
  )
}
```

**行为保持性核验**：
- 默认值：`HabitForm` 自带 fallback（见 §3.2），`initial={dataModel}` 后初值不变。
- `onSubmit={onConfirm}`：`HabitFormFields` 结构兼容 `Record<string, unknown>`（所有字段为 JSON 标量/数组）；按参数逆变，`(data: Record<string,unknown>) => void` 可赋给 `onSubmit: (fields: HabitFormFields) => void`，无需 cast。`onConfirm` 收到的对象即 `HabitForm` 提交的 `HabitFormFields`，与旧 adapter 经恒等映射后的结果逐字相同。
- 删除永远空 `[]` 的死 `useState`。
- 不加 `isDone` 分支（非目标）。
