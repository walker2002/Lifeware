## 概述

对 `/schedule` 时间盒管理页 6 项 UX 缺陷的集中修复，并在收尾时把 URL `/schedule` 硬切换到 `/timeboxes` 以消除歧义。SSOT = 新 `/timeboxes` 独立页（[023] A2 引入的 `ScheduleWorkspace`），主页（`/`）改为直接 redirect 到 `/timeboxes`，主页原本的 HomeBanner 4 按钮随之消失。

## 方案对比

### 主页改造（接 (3)+(6)）

| 方案 | 内容 | 优 | 劣 |
|---|---|---|---|
| **A1 整页 redirect** | `app/page.tsx` 改 `redirect('/timeboxes')`，删 AppProvider/Chat/ScheduleView/HomeBanner 全套 | 最简；4 按钮随 layout 自然消失；编译范围缩 | 失去主页 AI 助手 / 全局对话路径（如有依赖需另导出） |
| A2 保留 layout 仅替换主内容 | 保留 AppProvider + AppShell，`mainViewState.type==='schedule'` 渲染 `<ScheduleWorkspace />`；删 HomeBanner 4 按钮 | 不破坏 left-panel AI 助手 / 全局对话 | 主页仍是复合 layout，需保留跳转或显式导航 |

**选 A1**（贴合用户"主页指向 /schedule 即可"的意图；Chat/AI 助手能力是另一段故事）

### 中间列重叠渲染（接 (4)）

| 方案 | 算法 | 优 | 劣 |
|---|---|---|---|
| B1 仅颜色边框提示 | 重叠块加 border-error/border-warning，宽度不变 | 改动最小 | 重叠严重时仍视觉混乱 |
| **B3 区间扫描 + 列分配 + fallback** | 贪心装箱，totalCols = 同时间点 active 最大列数；>4 时仅标边框不缩宽 | 兼顾视觉与极窄场景 | 需 tooltip 列出冲突 |
| B2 贪心装箱无条件缩宽 | 同 B3 但无 fallback | 经典稳定 | 重叠链长时宽度细到字看不清 |

**选 B3**（推荐用户体验最稳）

### (1)+(5) 错误反馈

无方案分歧：补 try/catch + toast.error + needs_confirm 复用 AlertDialog 模式。

const handleAction = useCallback(async (

  action: 'start' | 'end' | 'cancel' | 'log',

  setActionSubmitting(true)

    const r = await transitionTimebox(timeboxId, action)

### §4 中间列重叠渲染（接 (4)）

`overlap-layout.ts`：

```ts
/**
 * @file overlap-layout
 * @brief 时间盒重叠区间扫描 + 列分配（B3 算法）
 *
 * 对 ScheduleEvent[] 按 startTime 升序，贪心装箱分配 column；
 * 同时间点 active 的最大 column 数 = totalCols；
 * totalCols > 4 → isOvercrowded=true（fallback 仅边框提示不缩宽）。
 *
 * itinerary 不参与（仅 kind='timebox'）。itinerary 在 timeline 仍按现状
 * 显示在底层（与 timebox 并排不同列）。
 */

export interface OverlapLayout {
  event: ScheduleEvent
  col: number
  totalCols: number
  isOvercrowded: boolean
}

export function computeOverlapLayout(events: ScheduleEvent[]): OverlapLayout[] {
  // 1. 仅 timebox 参与；itinerary 给定 col=0, totalCols=1
  const timeboxes = events.filter(e => e.kind === 'timebox')
  const others = events.filter(e => e.kind !== 'timebox')
  
  // 2. timebox 按 start 升序
  const sorted = [...timeboxes].sort((a, b) =>
    new Date(a.start).getTime() - new Date(b.start).getTime()
  )
  
  // 3. 贪心 column 分配
  const cols: number[] = []  // index = col, value = last event end (ms)
  const placements: Array<{ event: ScheduleEvent; col: number; endMs: number }> = []
  for (const ev of sorted) {
    const startMs = new Date(ev.start).getTime()
    const endMs = new Date(ev.end).getTime()
    let placed = -1
    for (let c = 0; c < cols.length; c++) {
      if (cols[c] <= startMs) {
        placed = c
        cols[c] = endMs
        break
      }
    }
    if (placed === -1) {
      placed = cols.length
      cols.push(endMs)
    }
    placements.push({ event: ev, col: placed, endMs })
  }
  
  // 4. 对每个 placement 算 totalCols（同时间点 active 的最大 col 索引 + 1）
  const totalColsByEvent = new Map<string, number>()
  for (const p of placements) {
    const pStart = new Date(p.event.start).getTime()
    const active = placements.filter(q =>
      new Date(q.event.start).getTime() <= pStart &&
      new Date(q.event.end).getTime() > pStart
    )
    const maxCol = Math.max(...active.map(a => a.col))
    totalColsByEvent.set(p.event.id, maxCol + 1)
  }
  
  // 5. 输出
  return [
    ...placements.map(p => {
      const totalCols = totalColsByEvent.get(p.event.id)!
      return {
        event: p.event,
        col: p.col,
        totalCols,
        isOvercrowded: totalCols > 4,
      }
    }),
    ...others.map(e => ({ event: e, col: 0, totalCols: 1, isOvercrowded: false })),
  ]
}
```
