## [003] AI 助手 Intent 优化： 基于 template_form 和 AI Markdown 的工作流优化

### 1. template_form 和 template_markdown

#### **必要性**

对于复杂的创建/规划操作（如创建一组 OKR + 关联 KR + 关联任务），通过读取附件生成Markdown 文档或者用户直接编辑 Markdown文档比逐步表单更适合用户协同编辑和 AI 辅助。这是对 `template_form` 路径的文档化扩展。

#### Domain manifest 的定义说明：

- 所有 template_form 字段均在 Domain 中明确定义（可以是脚本）
- 所有 template_markdown 模板均在 Domain 中定义（文件形式）

#### 标签页面切换

在主显示区中，template_form 和 markdown 页面可以相互切换，并在最后增加“下一步”的按钮，确认执行

#### 确认执行

确认执行，需要通过Nexus编排调用相关Domain 的钩子校验处理，并修改状态机，按照Nexus规范继续。





### 2.  template_mardown 的设计

- 当前OKR、项目/任务已有 template_markdown的定义，可以直接使用

- 需要增加设计 timebox 的Markdown模板



### **注意**

如果一个 Markdown 文件产生多个相互依赖的 StructuredIntent（如先创建 Objective，再创建引用该 Objective ID 的 KeyResult），Orchestrator 需要处理顺序依赖。

这个批处理逻辑在当前设计中没有定义，MVP 建议先限制 Markdown 工作流只产生单一 Domain 的操作，跨 Domain 批处理留后续版本。
