## [001] Tasks Domain 页面开发需求文档

---

### 0. 阅读指引

本文档是面向开发者的完整页面需求规格，覆盖三个页面的数据、交互、状态、边界情况。

所有写操作构造 `PrebuiltIntent` 进入 Rule Engine，只读查询直接调用 Repository

**开发范围**: 

Page:`/tasks`（任务树页面），/taskDetail， /threadDetail



---

### 1. 全局约定

#### 1.1 数据访问规则

```
只读（列表渲染、详情展示）
  → 直接调用 Repository，不走 Nexus 链路

写操作（创建、编辑、状态变更、删除）
  → 构造 PrebuiltIntent → Rule Engine → State Machine

禁止：
  → 页面组件直接调用 hooks.ts
  → 页面组件直接访问 Drizzle Schema
```

#### 1.2 路由结构

```
/tasks                      TaskTreePage      - 任务总览（主线列表 + 任务树）
/tasksDetail                 TaskDetailPage    - 任务详情与编辑
/threadsDetail               ThreadDetailPage  - 主线详情与编辑
```

#### 1.3 核心数据类型（页面层使用）

```typescript
// 主线（Thread）列表项
interface ThreadListItem {
  id: string
  name: string
  color: string           // CSS color string
  status: 'active' | 'paused' | 'completed' | 'archived'
  priority: 'critical' | 'high' | 'medium' | 'low' | null
  taskCount: number       // 该主线下的任务总数
  completedTaskCount: number
  startDate: string | null
  endDate: string | null
}

// 任务树节点
interface TaskTreeNode {
  id: string
  title: string
  status: 'todo' | 'planned' | 'in_progress' | 'completed' | 'archived'
  clarity: 'fuzzy' | 'scoped' | 'actionable'
  priority: 'critical' | 'high' | 'medium' | 'low'
  energyProfile: 'light' | 'deep' | 'admin' | 'creative' | 'reactive' | null
  schedulingConstraint: 'hard_deadline' | 'soft_target' | 'opportunistic' | 'recurring' | null
  tracking: 'none' | 'check_in' | 'log' | 'review'
  estimatedDuration: number | null  // 分钟
  dueDate: string | null
  parentId: string | null
  threadId: string | null
  childCount: number
  decomposition: 'atomic' | 'splittable' | 'splitting_in_progress' | 'decomposed' | null
  children?: TaskTreeNode[]         // 展开后填充
}

// 任务完整详情（详情页使用）
interface TaskDetail extends TaskTreeNode {
  description: string | null
  energyRequired: 'high' | 'medium' | 'low'
  captureMode: 'scheduled' | 'ad_hoc' | 'retrospective'
  complexity: string[]              // 非排他性标签数组
  actualDuration: number | null
  startDate: string | null
  endDate: string | null
  notes: string | null
  createdAt: string
  updatedAt: string
  completedAt: string | null
  acceptanceCriteria: string | null    // 【新增】 验收标准（自由文本）
  expectedOutput: string | null      // 【新增】预期产出物描述
}
```

#### 1.4 clarity 与 status 的视觉映射

所有页面统一使用以下视觉规范，不在单个组件内各自定义：

| clarity      | 视觉标识     | 含义提示文案     |
| ------------ | ------------ | ---------------- |
| `fuzzy`      | 灰色虚线圆点 | 想法模糊，待澄清 |
| `scoped`     | 橙色实心圆点 | 已有轮廓，待细化 |
| `actionable` | 绿色实心圆点 | 可执行           |

| status        | 视觉标识          |
| ------------- | ----------------- |
| `todo`        | 空心圆圈          |
| `planned`     | 蓝色圆圈          |
| `in_progress` | 蓝色旋转/填充圆圈 |
| `completed`   | 绿色勾选圆圈      |
| `archived`    | 灰色填充圆圈      |

页面Header，增加任务管理的 banner 图片和Action标题

---

#### 2.2 左侧面板：主线列表

##### 2.2.1 数据

调用 `ThreadRepository.findAll(userId, { excludeStatus: ['archived'] })`，返回 `ThreadListItem[]`，按以下规则排序：

- 先按 `status`：`active` > `paused` > `completed`
- 同 status 内按 `priority`：`critical` > `high` > `medium` > `low` > null
- 同 priority 内按 `updatedAt` 降序

##### 2.2.2 固定入口（列表顶部，不可移除）

```
[全部任务]      - 显示所有未归档任务，不区分主线
[无主线任务]    - 显示 thread_id IS NULL 且未归档的任务
```

##### 2.2.3 主线列表项

每个主线列表项包含：

```
[色块] 主线名称                    状态角标
       X个任务 · Y个已完成
```

- **色块**：4px 宽竖条，使用 `thread.color`
- **状态角标**：仅在 `status=paused` 时显示「已暂停」灰色角标，`active` 不显示
- **选中态**：背景色高亮，左侧色块加粗
- **右键菜单 / 长按菜单**（【MVP】）：
  - 编辑主线
  - 暂停主线 / 恢复主线（按当前 status 显示对应项）
  - 完成主线
  - 归档主线（仅 `completed` 状态可见）
  - 分隔线
  - 在此新建任务

##### 2.2.4 筛选栏（左侧面板底部）

【MVP】提供以下筛选维度，选中后右侧任务树同步过滤：

```
clarity:   [全部] [模糊] [有轮廓] [可执行]
status:    [全部] [待办] [计划中] [进行中] [已完成]
```

筛选状态存储在 URL query params（`?clarity=fuzzy&status=todo`），支持分享/刷新保持状态。

##### 2.2.5 创建主线入口

点击页面 Header 的「+ 创建主线」按钮：

- CNUI 最终使用 `thread-creation-card` CNUI Surface
- Page 打开右侧滑出抽屉，内含创建表单（见 2.5 节）

---

#### 2.3 右侧内容区：任务树

##### 2.3.1 数据加载

根据左侧选中项，调用对应 Repository 方法：

| 选中项     | Repository 调用                                              | 说明                      |
| ---------- | ------------------------------------------------------------ | ------------------------- |
| 全部任务   | `TaskRepository.findRoots(userId, filters)`                  | 返回 parentId=null 的任务 |
| 无主线任务 | `TaskRepository.findRoots(userId, { threadId: null, ...filters })` |                           |
| 某条主线   | `TaskRepository.findRoots(userId, { threadId, ...filters })` |                           |

初始只加载根节点（`parentId=null`），子节点在用户展开时按需加载（懒加载）。

##### 2.3.2 任务树节点

每个节点包含：

```
[展开箭头] [status圆圈] [clarity圆点] 任务标题     [priority角标] [due_date] [能量标签] [...]
```

- **展开箭头**：`childCount > 0` 时显示，点击展开/折叠子节点；`childCount == 0` 时显示占位空间（不显示箭头）
- **status 圆圈**：点击弹出状态变更快捷菜单（见 2.3.3）
- **clarity 圆点**：悬停时显示 tooltip 说明当前 clarity 含义
- **任务标题**：点击跳转 `/tasks/[id]`；双击进入行内编辑模式（仅标题，回车确认，Escape 取消）
- **priority 角标**：仅 `critical` 和 `high` 显示，分别用红色和橙色
- **due_date**：显示格式 `MM-DD`；逾期（< 今天）显示红色；3天内显示橙色
- **能量标签**：显示 `energy_profile` 的缩写图标（deep=⚡，light=☁，admin=📋，creative=✨，reactive=🔥）
- **`[...]` 更多菜单**：悬停时显示，点击弹出操作菜单（见 2.3.4）

子节点缩进 20px/层，最多支持视觉展示 5 层（更深层次折叠提示）。

##### 2.3.3 状态变更快捷菜单

点击 status 圆圈弹出，只显示合法跃迁项：

```
当前 todo:        → [计划中] [开始执行] [归档]
当前 planned:     → [开始执行] [回到待办] [归档]
当前 in_progress: → [标记完成] [暂停回待办] [归档]
当前 completed:   → [归档]
```

「标记完成」特殊处理：

- `tracking=none`：直接完成，无额外弹窗
- `tracking=check_in`：弹出「实际用时」输入框（分钟），填写后完成
- `tracking=log`：弹出「实际用时 + 产出描述（一句话）」，填写后完成
- `tracking=review`：跳转到任务详情页，在详情页底部的结构化总结区完成

##### 2.3.4 更多菜单（`[...]`）

```
在此下方新建子任务
提升为主线          （仅 parentId=null 且 threadId=null 时显示）
关联到主线...       （弹出主线选择器）
移出主线            （仅 threadId !== null 时显示）
---
编辑任务
复制任务
---
归档任务
删除任务            （二次确认：「删除后子任务将成为独立任务，确认删除？」）
```

##### 2.3.5 快速添加任务（行内创建）

页面 Header「+ 快速添加任务」按钮，或在任务树底部点击「+ 添加任务」：

- 在列表末尾插入一行输入框
- 输入标题，回车创建（`captureMode='ad_hoc'`，`clarity` 系统自动计算）
- 创建后任务出现在列表，同时显示「AI 正在分析标签...」loading 状态 1-2 秒后刷新标签

##### 2.3.6 空状态

| 场景           | 显示内容                                               |
| -------------- | ------------------------------------------------------ |
| 主线下无任务   | 「这条主线还没有任务」+ 「+ 在这里添加第一个任务」按钮 |
| 筛选后无结果   | 「当前筛选条件下没有任务」+ 「清除筛选」链接           |
| 无主线任务为空 | 「所有任务都已关联到主线了」                           |

#### 3.2 A 区：任务信息（编辑区）

##### 3.2.1 数据加载

调用 `TaskRepository.findById(id)` 返回 `TaskDetail`。

##### 3.2.2 字段与编辑行为

所有字段采用「点击编辑」模式（inline editing），非 form 表单提交。每个字段独立保存，失焦或回车触发 `PrebuiltIntent`。

| 字段                 | 展示形式                     | 编辑形式                                            | 必填 | 说明                                                |
| -------------------- | ---------------------------- | --------------------------------------------------- | ---- | --------------------------------------------------- |
| title                | 大号标题文本                 | 单行 input                                          | ✅    | 最多 100 字符                                       |
| description          | Markdown 渲染                | Markdown 编辑器                                     | ❌    | 支持基础 MD 语法；为空时显示「点击添加描述...」占位 |
| priority             | 彩色角标                     | 下拉选择                                            | ✅    | critical/high/medium/low                            |
| energyRequired       | 图标+文字                    | 三段选择（高/中/低）                                | ✅    | 区别于 energy_profile（用户感知的能量需求）         |
| energyProfile        | 图标+文字                    | 下拉选择                                            | ❌    | AI 推荐值高亮显示                                   |
| schedulingConstraint | 文字标签                     | 下拉选择                                            | ❌    | AI 推荐值高亮显示                                   |
| tracking             | 文字标签                     | 下拉选择                                            | ✅    | 显示各选项的完成要求说明                            |
| captureMode          | 文字标签                     | 下拉选择（不常用，放收起区）                        | ✅    |                                                     |
| estimatedDuration    | 「预估 X 分钟」              | 数字 input（分钟）或时长快选（30/60/90/120/180min） | ❌    |                                                     |
| dueDate              | 日期文字（颜色规则同树视图） | Date picker                                         | ❌    |                                                     |
| startDate / endDate  | 日期文字                     | Date picker                                         | ❌    | 折叠在「更多信息」中                                |
| threadId             | 主线名称+色块                | 主线选择器弹窗                                      | ❌    | 可清除关联                                          |
| tags                 | 标签列表                     | 标签输入                                            | ❌    |                                                     |
| notes                | 纯文本                       | textarea                                            | ❌    | 折叠在「更多信息」中                                |

**「更多信息」折叠区**：默认收起，包含 startDate、endDate、captureMode、notes、recurrence。

**未实现字段占位**（必须有 UI 占位，但功能不可用）：

- `acceptanceCriteria`：「验收标准」字段，灰色占位文字「即将支持」
- `expectedOutput`：「预期产出」字段，同上

##### 3.2.3 编辑保存逻辑

```
用户修改字段 → 失焦 / 回车
→ 构造 PrebuiltIntent { action: 'editTask', fields: { id, [changedField]: newValue } }
→ 乐观更新 UI（立即反映修改）
→ 后台提交 Intent
→ 失败时回滚 UI + 显示错误 Toast
```

**title 的特殊处理**：title 修改后，系统后台自动重新计算 clarity，B 区认知面板刷新。

#### 3.3 B 区：系统认知面板（只读）

**标题**：「系统认知」（System Intelligence）

**关键设计原则**：B 区是系统对任务的「理解报告」，用户不能直接修改其中任何值，只能通过修改 A 区字段间接影响。

##### 3.3.1 认知清晰度（clarity）

```
┌─────────────────────────────────┐
│ 认知清晰度                       │
│                                  │
│  ●──────●──────○                 │
│  模糊   有轮廓  可执行            │
│                                  │
│  当前：有轮廓                    │
│  缺少：预估时长                  │
│  → 填写「预估时长」后可升级       │
└─────────────────────────────────┘
```

- 三段进度条，当前状态高亮
- 显示「升级至下一级需要什么」的明确提示
- 当可升级时（所有条件满足），显示「✨ 已可执行」绿色提示，无需用户操作（系统自动）

##### 3.3.2 复杂度标签（complexity）

展示当前 AI 识别到的 complexity 标签数组，每个标签附带说明文字：

```
复杂度标签
[多步骤]  [需探索]
```

- 灰底标签，悬停显示该标签的定义
- 底部显示「【预留】AI 语义分析中...」（针对非规则计算的标签）

##### 3.3.3 拆分建议（decomposition）

```
可拆分状态：
┌─────────────────────────────────┐
│ 拆分建议                         │
│ 预估时长 > 2小时，建议拆分        │
│ [AI 建议拆分方案]  ← 触发 splitTask │
└─────────────────────────────────┘

已拆分状态：
┌─────────────────────────────────┐
│ 拆分建议                         │
│ 已拆分为 3 个子任务，完成 1/3     │
└─────────────────────────────────┘
```

- `splittable`：显示「建议拆分」+ 「AI 建议拆分方案」按钮（【CNUI】触发 `task-split-card`；【MVP】触发 AI 细化对话占位弹窗）
- `atomic`：显示「可直接执行，无需拆分」
- `splitting_in_progress`：显示进度
- `decomposed`：显示「已完全拆分」

##### 3.3.4 AI 推荐标签对比

展示用户设置值 vs AI 推荐值的对比（当两者不同时显示）：

```
AI 推荐对比
energy_profile:  你设置「轻量」  AI 推荐「深度工作」  [采纳建议]
```

「采纳建议」构造 PrebuiltIntent 更新对应字段。

#### 3.4 C 区：子任务列表

##### 3.4.1 展示

显示 `parentId = 当前任务ID` 的直接子任务，不递归展示孙任务。

每个子任务行：

```
[status圆圈] [clarity圆点] 子任务标题    [priority] [due_date] [→ 详情]
```

排序：`in_progress` > `planned` > `todo` > `completed`，同 status 内按 priority 排序。

##### 3.4.2 新建子任务

列表底部「+ 添加子任务」行内输入，回车创建，逻辑同任务树的快速添加。

新建时继承父任务的：

- `tracking`（默认继承，可修改）
- `threadId`
- `priority`（继承，可修改）

##### 3.4.3 子任务完成率

### 6. 边界情况与错误处理

| 场景                                  | 处理方式                                                     |
| ------------------------------------- | ------------------------------------------------------------ |
| 修改任务不存在（ 404）                | 显示「任务不存在或已删除」+ 返回按钮                         |
| 修改主线不存在（ 404）                | 同上                                                         |
| 子任务数量 > 50                       | 默认只加载前 20 条，底部「加载更多」                         |
| 任务树层级 > 5 层                     | 第 6 层起折叠，显示「展开更深层级（X 个任务）」              |
| 删除有子任务的父任务                  | 二次确认弹窗：「删除后，X 个子任务将变为独立任务，确认继续？」 |
| 设置子任务 due_date > 父任务 end_date | 行内警告：「此日期晚于父任务结束日期（X月X日）」，允许保存但警告 |
| 离线状态下尝试写操作                  | Toast 提示「当前离线，操作将在联网后同步」（预留，MVP 可只做提示） |
